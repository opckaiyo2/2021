### **pythonでは1行コメントは行頭に「#」ですが複数行コメントアウトするときに「\"""」で囲います。**  
### **下のように記述するとコメント初め「\"""」の行頭を「#」に切り替えることで,**
### **簡単に複数行コメントを切り替えられます。**
例  
この時はprint文は実行される

```
#"""  
print("test")  
#"""
```

この時はprint文は実行されない

```
"""  
print("test")  
#"""  
```

### **PID制御について**  
PIDとは比例、微分、積分制御を同時におこなうことで正確な制御を行う制御です  
Kpは比例制御 `誤差が多ければ多いほど操作量を増やす`  
Kiは積分制御 `誤差のある状態が長い時間続けばそれだけ操作量の変化を大きくして目標値に近づけようとする`  
Kdは微分制御 `急激な操作量の変化が起こった場合その変化の大きさに比例した操作を行うことでその急激な操作量に抗しようとする役目を果たす。`

## 書き方できをつけている事
***
### **try文**
エラーが発生しそうな場所はtry文でエラーを検出してます。  
機体の航行に影響する重大なエラーはループを抜けるように設計しています。  
また、通信をはじめる時にエラーが発生する場合があります。  
通信開始時のエラーは処理を再試行するように、  
ループを抜ける時は必ずどのファイルのどの関数でどんなエラーが発生したか記述しています。
<br><br>

### **ループを抜けるとき**
ループから抜ける時は必ずモータを止めること。もし、止めなかったらモータが周り続ける。  
その時は緊急用スイッチを抜いてもよい。
<br><br>

### **処理が重いコード**
処理が重いコードはmultiproceeを使ってコアを分割して処理をしています。  
しかし、raspiはコアが4しかないためコアを分割して処理を軽くできるのはprocessが4つ程と考えられる。
cpu使用率やコアの使用率を検証していないので正確ではないかもしれないが、  
process数は4つに制限している。
<br><br>

### **コンフィグファイル**
機体の動きを切り替えるときコードを編集したりコメントアウトするのは手間が掛かるため、  
コンフィグファイルを採用している。  
コード実行するときにコンフィグファイルが読み込まれる。  
コンフィグファイルの書き方はファイルに記述している。

## わかりにくい構文
***
### **try**
tryは例外が発生した際にプログラムを終了させずに例外後に処理を行えます。  
下の例文のExceptionはすべての例外に対し反応します。  
ここを例えばKeyboardInterruptに変更するとキーボードインプットで発生した例外に対し反応します。  
プログラムをctrl+cで終了した時などに反応します。

```python
try:
    例外が出る可能性がある処理
    例外の種類によって後の挙動を変えたい処理
except Exception as e:
    例外後の処理
```

### **class**
クラスを理解する上で、欠かせないのが、「クラス」、「インスタンス」で、それぞれに関連があります。  
クラスは料理のレシピです。私たちはレシピを定義します。  
レシピを定義したら実際に料理を作成します。レシピから作成された料理をインスタンスと呼びます。  
プログラム上ではクラスだけでは使用できず必ずインスタンスを作成します。

下の例文を見てみましょう。`Test`という名前のクラスが定義されています。
`__init__`はクラスがインスタンス化されたときに必ず実行される関数です。
この関数をコンストラクタと呼びます。

`__init__`の中で`ins`という名前の変数が定義されています。  
しかし変数の前に`self`がついています。これはインスタンス変数と言います。
インスタンス変数はインスタンスごとに違う値を持ちます。
インスタンス変数の反対をクラス変数を呼びます。
クラス変数はselfをつけずに定義すれば良いだけです。

```python
class Test():
    def __init__(self):
        self.ins = 0
        print("クラスからインスタンスが作成されました。")

    def show(self):
        print(self.ins)


if __name__ == "__main__":
    test1 = Test()
    print(test1.ins)
    test1.show()
```

実行結果

```
クラスからインスタンスが作成されました。
0
0
```
